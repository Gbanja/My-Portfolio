Animating with CSS

By GbanJah Campbell Jr. • August 3, 2025

Animations breathe life into your web pages—guiding user attention, providing feedback, and adding that extra bit of polish. In this post, we'll explore the core techniques for creating smooth, performant CSS animations and see practical examples you can drop straight into your own projects.

1. Why CSS Animation?

  • No JavaScript required: Pure‐CSS animations offload work to the compositor thread, keeping interactions smooth.
  • Declarative control: Define keyframes and transitions in your stylesheet, making them easy to tweak.
  • Broad support: Modern browsers fully support @keyframes and transition.

2. Transitions vs. Keyframes

Transitions let you animate between two states—ideal for hover effects, focus styles, or simple property changes:

button {
  transition: background 0.3s ease, transform 0.3s ease;
}
button:hover {
  background: var(--color-accent);
  transform: translateY(-3px);
}

Keyframes give you full control over a sequence of states over time—perfect for pulsing glows, slides, rotations, and more:

@keyframes pulseGlow {
  from { box-shadow: 0 0 8px var(--color-cta); }
  to   { box-shadow: 0 0 20px var(--color-accent); }
}
.button {
  animation: pulseGlow 3s ease-in-out infinite alternate;
}

3. Best Practices

  1. Animate transform & opacity: These are the cheapest, most performant properties.
  2. Avoid layout thrashing: Don’t animate width or margin if you can use transform: scale().
  3. Keep it subtle: Overly flashy animations can be distracting—use them to reinforce, not overwhelm.

4. Real-World Examples

Hover Glow Button

.button {
  position: relative;
  transition: transform 0.3s;
}
.button:hover {
  transform: translateY(-4px);
}
@keyframes glow {
  0% { box-shadow: 0 0 8px var(--color-cta); }
  100% { box-shadow: 0 0 16px var(--color-accent); }
}
.button {
  animation: glow 4s ease-in-out infinite alternate;
}

Image Lightbox Fade

.lightbox-overlay {
  opacity: 0;
  transition: opacity 0.4s ease;
}
.lightbox-overlay.show {
  opacity: 1;
}

5. Performance Tips

  • Use the Chrome DevTools “Rendering” tab to highlight painted areas and verify GPU compositing.
  • Limit simultaneous animations on the same element—too many can drop frames.
  • Consider will-change sparingly to hint at upcoming animations.

Conclusion

CSS animations are an essential tool for modern web development. From micro-interactions to full-page hero animations, they can elevate your user experience without heavy scripting. Experiment with the examples above, and don’t forget to keep an eye on performance as you add more motion to your sites.
